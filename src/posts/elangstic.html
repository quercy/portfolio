This article will show you how to connect Elasticsearch's API to Angular for blazingly fast full-text all-field searches; the example application is a type-to-search-engine.

Introduction to Angular

If you aren't familiar with Angular, and can handle a bit of dryness, AngularJS' own introduction is great. If you like the idea of a tour of a pre-built application with concepts being explained on the way, go grab AngularJS in Action. If you are slightly familiar with Angular but don't know, for example, the difference between a provider, factory, and service -- Tyler McGinness is a great resource (but keep in mind that while a factory is a service - they are all singletons!). If you don't know Javascript, I'm not sure what you're doing here.

Introduction to Elastic/Elasticsearch

Elasticsearch deserves an entire book written about it. Thankfully, the beautiful people at Elastic wrote one for you.. If you're short on time, the key points are this:

Elasticsearch is...

  • "A real-time distributed search and analytics engine."
  • "A distributed real-time document store where every field is indexed and searchable."
  • "A distributed search engine with real-time analytics."
  • "Capable of scaling to hundreds of servers and petabytes of structured and unstructured data"

But for our puropses, Elastic search is...

  • Really, really, really fast at full text searches over many fields
  • Really, really easy to throw data at
  • Really easy to set up
  • Not as easy to get started querying with Angular
  • Exposes its interface via a RESTy API
  • Utilizes a JSONy document store
