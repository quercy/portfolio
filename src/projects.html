Personal Projects

SomethingGUD Mobile

Description

My senior project to complete my CS degree. It was a full-stack prototype for a cross-platform mobile shopping cart application for a (now-defunct) literal farm-to-table delivery company. The app allowed users to register, select a date for delivery, view categories of foods, add items to the cart, and check out.

Retrospective

This project went from design to deployment/presentation in about 3 days, so there were necessary constraints on the scope; however, the short timeline was a good exercise in crunch-work, and was good practice to picking up new technologies lightning-quick. Given more time, I would have refactored to use a JS framework like Angular, Vue, etc.

Technologies used

  • PHP
    • PDO extension
    • Slim - framework for the REST API
  • MySQL
  • JavaScript
    • jQuery + jQuery Mobile
  • git/Github

This site

I started with a simple sketch of the layout in Photoshop, and then mocked it up in HTML/CSS. I then made a build system using Gulp to compile, minify, and concatenate the JS and SASS, using Node to run the server, but that quickly got unweildy for such a small site. I stumbled on Metalsmith, a static site generator based on plugins and file-based pages and templates - which is perfect in terms of speed and ease. I added file watchers to automatically compile the site and autoreload any open pages for rapid develoment, and a Makefile to quickly deploy. This made it easy to switch from EC2 to S3 hosting for the site, simply by modifying the deploy script; nothing else had to change.
