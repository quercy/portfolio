This site

I started with a simple sketch of the layout in Photoshop, and then mocked it up in HTML/CSS. I then made a build system using Gulp to compile, minify, and concatenate the JS and SASS, using Node to run the server, but that quickly got unweildy for such a small site. I stumbled on Metalsmith, a static site generator based on plugins and file-based pages and templates - which is perfect in terms of speed and ease. I added watchers to automatically compile the site and autoreload any open pages for rapid develoment, and a Makefile to quickly deploy.

What I learned

Build tools like Gulp can be a time hole, even if they are cool. Don't rely on documentation. There is a place for static websites - they are incredibly fast. Are you going to blame IE for breaking your design? Try W3C's HTML & CSS validators.

Technologies used

  • Javascript
    • Node/npm - package management, building, scripting, deploying
    • jQuery with Velocity - animation, DOM manipulation
  • SASS
  • bash - deploy script, Makefile
  • Photoshop - mockups/logo
  • git/Github
